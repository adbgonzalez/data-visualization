# Power BI: introdución

## 1. Que é Power BI

**Power BI** é unha ferramenta de análise e visualización de datos desenvolvida por Microsoft. Permite conectar datos procedentes de distintas fontes, transformalos, modelalos e crear informes interactivos que despois se poden compartir con outras persoas.

En termos prácticos, Power BI serve para converter táboas e rexistros dispersos en gráficos, indicadores e paneis que axudan a comprender mellor a información.

Nun fluxo de traballo habitual con Power BI adoitan aparecer estas tarefas:

- conectar cunha fonte de datos
- limpar ou transformar os datos
- relacionar táboas
- crear medidas e cálculos
- deseñar visualizacións
- publicar e compartir o informe

---

## 2. Elementos principais de Power BI

Ao falar de Power BI convén distinguir varios compoñentes, porque non todo sucede no mesmo lugar.

### Power BI Desktop

É a aplicación de escritorio que se usa para construír informes. Nela realízanse a maior parte das tarefas de conexión, transformación de datos, modelado e deseño visual.

É o punto de partida máis habitual cando se comeza a traballar coa ferramenta.

### Power BI Service

É a parte web da plataforma. Utilízase para publicar informes, compartilos, actualizar datos e crear dashboards a partir dos elementos publicados.

Nun contexto educativo, pódese entender como o espazo no que os informes deixan de ser un ficheiro local e pasan a estar dispoñibles para o seu consumo e seguimento.

### Dataset, informe e dashboard

En Power BI aparecen con frecuencia tres conceptos que é importante non confundir.

- **Dataset**: conxunto de datos xa cargado e preparado para a análise.
- **Informe**: colección de páxinas con gráficos, filtros e elementos interactivos.
- **Dashboard**: panel resumo, normalmente dunha soa páxina, que reúne visualizacións clave para unha consulta rápida.

Un informe adoita servir para explorar información con máis detalle, mentres que un dashboard está máis orientado a ofrecer unha visión xeral.

## 3. Fluxo básico de traballo

Power BI adoita empregarse seguindo unha secuencia bastante clara.

### 3.1. Conexión aos datos

O primeiro paso consiste en importar ou conectar unha fonte de datos. Power BI pode traballar con ficheiros locais, bases de datos, servizos na nube e outras aplicacións.

Exemplos frecuentes:

- ficheiros `Excel`
- ficheiros `CSV`
- bases de datos relacionais
- servizos de Microsoft
- fontes web

### 3.2. Transformación dos datos

Antes de visualizar, moitas veces é necesario limpar os datos. Isto inclúe tarefas como:

- renomear columnas
- cambiar tipos de datos
- eliminar filas baleiras ou erros
- dividir ou combinar columnas
- filtrar rexistros

Estas operacións adoitan realizarse no editor de consultas, coñecido tamén como **Power Query**.

### 3.3. Modelado

Unha vez cargados os datos, pódense crear relacións entre táboas e definir cálculos.

Nesta fase son especialmente importantes:

- as relacións entre táboas
- as xerarquías
- as columnas calculadas
- as medidas

O modelo de datos inflúe directamente na calidade da análise. Se as relacións son incorrectas ou os tipos de dato non están ben definidos, os resultados poden ser confusos ou erróneos.

### 3.4. Creación do informe

Co modelo preparado, xa se poden engadir visualizacións á área de deseño do informe.

Nesa fase escóllense:

- os campos que se van representar
- o tipo de gráfico
- os filtros
- o formato visual
- a disposición dos elementos

### 3.5. Publicación e compartición

Por último, o informe pódese publicar para que outras persoas o consulten desde a web. A partir de aí tamén se poden construír dashboards e configurar actualizacións.

---

## 4. Visualizacións e interactividade

Un dos puntos fortes de Power BI é que permite crear informes interactivos. Isto significa que o usuario pode filtrar, destacar ou explorar os datos sen modificar o conxunto de datos orixinal.

Entre as visualizacións máis habituais están:

- gráficos de barras
- gráficos de liñas
- gráficos circulares
- mapas
- táboas e matrices
- tarxetas de KPI
- segmentadores de datos

Ademais, os elementos dunha páxina poden interactuar entre si. Por exemplo, ao seleccionar unha categoría nun gráfico de barras, o resto de visualizacións poden actualizarse para mostrar só a información relacionada con esa selección.

Esta capacidade de exploración fai que Power BI sexa moi útil tanto para análise descritiva como para seguimento de indicadores.

---

## 5. Power BI e a análise empresarial

Power BI emprégase moito en contornos de intelixencia de negocio porque permite pasar de datos operativos a información útil para a toma de decisións.

Por exemplo, nunha organización pódese usar para:

- seguir vendas e ingresos
- analizar custos
- controlar inventario
- supervisar tempos de resposta
- monitorizar indicadores de rendemento

Neste contexto, Power BI non é só unha ferramenta para facer gráficos, senón unha plataforma para construír informes analíticos que axudan a interpretar o funcionamento dunha actividade ou servizo.

---

## 6. Vantaxes e limitacións

Como calquera ferramenta, Power BI ten puntos fortes e tamén algunhas limitacións que convén coñecer.

### Vantaxes

- interface visual bastante accesible
- boa integración con ferramentas de Microsoft
- grande variedade de conectores
- capacidade para crear informes interactivos
- boas opcións para compartir resultados

### Limitacións

- require certa aprendizaxe para modelar ben os datos
- algunhas análises dependen de comprender medidas e contexto de filtro
- o rendemento pode verse afectado con modelos grandes ou mal deseñados
- non substitúe un bo deseño de datos nin unha interpretación correcta dos resultados

---

## 7. Ideas clave para comezar

Ao empezar con Power BI convén quedar con varias ideas fundamentais:

1. Un informe visual bo depende de datos ben preparados.
2. Modelar os datos correctamente é tan importante como escoller un gráfico axeitado.
3. A interactividade é unha das grandes vantaxes da ferramenta.
4. Un dashboard serve para resumir; un informe serve para explorar con máis detalle.

Power BI resulta especialmente útil cando se quere combinar análise, visualización e compartición nun mesmo fluxo de traballo.
